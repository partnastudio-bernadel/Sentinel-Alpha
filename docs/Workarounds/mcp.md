# Model Context Protocol (MCP) Integration Limitations & Design Mitigations

The following table summarizes the engineering limitations surfaced during the implementation and deployment of MCP servers (specifically ForexFactory Scraper and Alpha Vantage) in our multi-agent economic surprise ingestion pipeline, along with the implemented mitigations.

| MCP Server | Category | Limitation Description | Implemented Mitigation / Solution |
| :--- | :--- | :--- | :--- |
| **ForexFactory Scraper** | Playwright/Scraper Startup & Render Latency | ForexFactory is asset-heavy. Launching headless Playwright Chromium and loading the calendar state frequently exceeded the default 5-second timeout (`SCRAPER_TIMEOUT_MS`). | Increased `SCRAPER_TIMEOUT_MS` to `30000` (30s) in [`.env.local`](file:///d:/PartnaStudio/sentinel/stack/FinRobot-IntentChain/sentiment/.env.local). Configured subprocess spawning to inherit environment variables, and enabled `headless=False` support for headed visual troubleshooting. |
| **Alpha Vantage** | API Rate Limiting & Response Shifting | Alpha Vantage free tier is limited to 25 requests/day. Upon exhaustion, the server returns an `"Information"` key explaining the limit instead of expected `"data"` array keys, causing key lookup crashes. | Implemented dynamic response parsing to detect the `"Information"` rate-limit warning. Added safe baseline fallbacks (e.g., standard deviation of `1.141` for CPI) to allow pipeline execution to proceed uninterrupted. |
| **Alpha Vantage / General** | Parameter Type Mismatches | In-context LLM tool calls pass JSON payloads where numeric parameters (like `window`) may serialize as strings (e.g., `"12"`). Slicing or addition (`records[:window+1]`) triggers a Python `TypeError`. | Added explicit type sanitization (e.g., `window = int(window)`) at the start of helper tool wrappers inside [`mcp_tools.py`](file:///d:/PartnaStudio/sentinel/stack/FinRobot-IntentChain/sentiment/functions/tools/mcp_tools.py) before performing array indexing operations. |
| **Orchestration Framework** | Nested Chat Response Swallowing | AutoGen's standard `register_nested_chats` handler returns `(True, summary)`, halting the parent agent's reply chain and preventing the Chief Macro Economist from executing its final report aggregation. | Implemented a custom nested chat callback (`custom_macro_nested_chat_reply`) that retrieves the data, silently appends it to the sender's history, and returns `(False, None)` to force final orchestrator aggregation. |
| **Orchestration Framework** | Summary Extraction on Short Turns | Under short turn limits (`max_turns: 2`), chats terminate on raw tool response payloads rather than conversational messages. AutoGen's default `"last_msg"` summarizer evaluates to an empty string. | Customized the nested chat handler to scan the sub-agent conversation history in reverse to extract the last non-empty content string (the tool response text) for inclusion in the final summary. |
